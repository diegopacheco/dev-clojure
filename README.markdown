## Purpose

This group is focused on learning Clojure. Our goal is create a small Java BANK system that calls <br />
some clojure functions to do some math processing with huge concurrency using STM. <br />

## WHY Clojure ?

* Java has lots and lots synatx obesity <br />
* OOP is not the holly grail and don't awnser all questions <br />
* Clojure works really well with concurrency, read STM <br />
* Clojure you have one function for 10 types of data, java you have 10 to 10 <br />
* Clojure is Functional, like LISP BUT much better <br />
* Clojrue Runs on the JVM, so you can use all java frameworks like Spring, Hibernate, etc... <br />
* Work with Clojrue will make you a better java developer <br />
* Open your mind, new paradigms <br />
* Clojure enables you think more about the problemn rather than in the syntax, frameworks, JEE, other heavy craps... <br />
* Twitter is using Clojure in production <br />
* Horoku supports Clojures at the CLOUD => http://blog.heroku.com/archives/2011/7/5/clojure_on_heroku/ <br />

More... Read it here: http://clojure.org/rationale <br />
